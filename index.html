Which Linux Distro?

What is a Distro?

Simply put, a Linux distro is a collection of choices. It is a complete operating system that enables a PC (or similar computing device) to boot from a BIOS screen to a user or admin command shell.

Collection of Choices:

  • Use Case: (ie, Desktop distro, Gaming, Server Rack, Cloud, Repair distro, etc.)
  • Intended Audience (noob friendly, administrator, admin, programmer, etc.)
  • Licensing Philosophy
  • Kernel Series
  • Which Software?
  • Release Cycle
  • Packaging System
  • Startup Method
  • Independent Distros
  • Protest Distros

History and the Parent Distros

In the beginning, around the summer of 1991, Linus announced he had developed a kernel for the x386 and was looking for help testing it. He made this request on a newsgroup, or a public text-based forum of the day. Within a few years several people were providing a collection of GNU system software with Linus's kernel on disk images. These became the first Linux distributions; most of these are gone now. But a few remained and became the parents of all current Linux distributions.

Linux itself is not an complete operating system--it is a kernel only. But a distribution of linux includes the kernel with a collection GNU software that utilizes a kernel to start up a system and run it. As of 1991, GNU had not completed testing on a free kernel. GNU had been working on a much more complicated design called a micro kernel called HURD, and was not nearly ready for production use yet.
